Glossary

Every technical word this course uses, in plain English. When a lesson introduces a new word, it's also defined here.

WordPlain-English meaning
BackupA complete copy of the device's storage, saved somewhere safe, so you can restore it if a change goes wrong.
Boot / bootingEverything that happens between power-on and Linux being fully up and running.
BootloaderThe very first tiny program that runs at power-on. Its job is to find Linux and start it. Like a car's starter motor.
Brick / brickedA device that no longer starts properly after a bad change.
CPUThe processor — the "brain" that runs all the software.
ConfigurationThe device's settings: passwords, network names, ISP data, and so on.
EthernetThe common wired-network connection your computer or router uses.
Flash storageThe device's permanent storage (like an SSD). Keeps data even with the power off.
ImageA complete copy of the storage or operating system, saved as one file.
LinuxThe operating system running inside this device — the same family that runs many servers and phones.
Operating systemThe main software that manages everything and runs the other programs.
Optical fiberA thin glass strand that carries Internet data as pulses of light.
PartitionOne labelled section of storage. Storage is split into several: bootloader, Linux, settings, and so on.
RAMFast temporary memory. Linux is copied here to run, and it's wiped when power is removed.
Reverse engineeringWorking out how something works by examining it, without the maker's design documents.
RootThe all-powerful user account in Linux that can change almost anything on the system.
SSHA secure way to reach and control a Linux system over a network.
TelnetAn older, unencrypted way to reach a Linux system over a network.
UARTA simple wired connection engineers use to watch and talk to the device as it starts up. Covered in Lesson 3.
U-BootA bootloader used by many embedded Linux devices (see Bootloader).
