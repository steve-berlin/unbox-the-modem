Part 8 — Linux Processes and Services

How a running Linux system keeps dozens of programs going at once.

TL;DR

A process is just a program that is currently running.

Every process gets its own number, called a PID, so the system can tell them apart.

The very first process (called init) starts when Linux boots and then starts everything else.

Programs that stay running quietly in the background are called services (or daemons). On a device like this one, services are what keep your fiber connection alive.

A program vs a process

A program is a file sitting in storage, doing nothing. A process is that same program after you start it — now it is alive in memory, using the CPU, maybe reading the network.

The same program can run as several processes at once. Think of a recipe (the program) versus three cooks each making that recipe at the same time (three processes).

Every process has a number (the PID)

When Linux starts a process, it hands it a PID — short for "process ID", just a plain number. No two running processes share a number at the same time, so the system can point at exactly one of them to check on it or stop it.

The first process to start always gets PID 1. That one is special.

init: the first program, the parent of all others

When Linux finishes loading, it starts a single program known as init (short for "initialise"). This is PID 1. Its job is to start everything else the system needs, in the right order.

Because init starts the others, and those may start still more, every process on the machine can be traced back up to init. It is the root of a family tree.

What is a service (daemon)?

Most programs you use run for a moment and then quit — you open them, they do a job, they stop. A service is different: it is a program meant to keep running for as long as the device is on, waiting quietly to do its job whenever it is needed.

The old word for a service on Linux is daemon (pronounced "demon"). You will see both words. Many daemon names even end in the letter d — for example a logging daemon might be called logd.

On your fiber device, the programs that keep the connection up, hand out network settings, and answer the management web page are all services. They were never opened by a person — they start on their own and stay running.

How services start automatically at boot

You never click anything to start these background programs. When the device powers on, init reads a list of services it is supposed to bring up, and starts each one. If a service later crashes, init can often start it again automatically. That is why a router or fiber device can sit unattended for months and keep working.

Looking at the list of what's running

On any Linux system you can ask "what is running right now?" Two common read-only tools answer that:

  • ps — prints a one-time list of processes: their PID, who owns them, and the program name.
  • top — shows the same kind of list but keeps refreshing, sorted by which processes are busiest. Press q to quit it.

Both only report. They do not change anything, which makes them safe to explore with.

Examples

  • Your web browser: a program on disk until you open it. Open it and it becomes a process with a PID. Open it twice and you have two processes.
  • A clock in the corner of a screen: a tiny service. Nobody re-launches it every minute; it just keeps running.
  • The fiber connection on this device: kept alive by a background service. If it stops, init can restart it — no human needed.
  • A reading of ps might show a line like 1 root init: PID 1, owned by the root user, running the program init. That single line is the top of the whole tree.

Quick quiz

1. What is a "process"?

2. Which process starts first and starts all the others?

3. What is a "service" (daemon)?

Try it yourself (safe)

Pick one of these. Neither changes anything.

  1. On any Linux PC you already own, open a terminal and type ps then Enter. You will get a short list. Find the columns that show the PID number and the program name.
  2. Or, with no computer at all: write two or three sentences in your own words explaining what a service is and why it does not need a person to start it.

Do this on a normal Linux PC or your own account, not on the fiber device. ps and top only read — but stay with read-only tools while you are learning.

Self-check: in a line of ps output, what does the first number usually mean, and what makes PID 1 special?

The first number is the process's PID — its unique ID for as long as it runs. PID 1 is special because it is init, the first process started at boot and the parent that every other process descends from.

Recap

  • A process is a running program; a program on disk is not yet a process.
  • Each running process has a unique number, its PID.
  • init is PID 1 — the first process, and the parent of all the others.
  • Services (daemons) are background programs that keep running the whole time the device is on.
  • Services start automatically at boot; init can restart them if they crash.
  • ps and top list what is running, and only read — they change nothing.
