Part 6 — Flash Storage and Partitions

Where the device keeps everything, and how that space is divided up.

TL;DR

The device has permanent storage — memory that keeps its contents even when the power is off. Engineers call this kind of storage flash.

That storage is not one big blob. It is divided into labelled sections called partitions, each holding one kind of thing: the bootloader, the operating system, the settings, and sometimes a spare recovery copy.

The split lets each section be updated or repaired on its own — and it means erasing the wrong section can stop the device from starting.

What is permanent storage?

Permanent storage is memory that remembers its contents with the power off — the same idea as the drive in a laptop. On small devices like this one, that storage is a flash chip: a single small chip soldered to the board.

You will sometimes hear the two common types named NAND and NOR. For this course the difference does not matter much — both are permanent storage. What matters is what is stored and how it is arranged.

Not the same as RAM. RAM (from Part 5) is fast, temporary working memory that is wiped the moment power is lost. Flash keeps its contents. Linux lives in flash and is copied into RAM to run.

What is a partition?

Imagine one long shelf divided by dividers into labelled boxes. Each box has a name and holds one kind of item, and you can tidy or replace one box without disturbing the others. Partitions are exactly that: the storage is split into labelled sections, and each section holds one part of the system.

What each section usually holds

On many embedded devices the sections do these jobs. The exact names differ from device to device, so treat these as the usual roles, not fixed facts:

SectionWhat it holds
BootloaderThe small start-up program that loads the operating system (Part 5).
Operating systemLinux itself and the programs that run on it.
SettingsYour configuration: passwords, network names, ISP details. This survives a restart.
Recovery copyOn some devices, a spare copy used to repair a broken operating system.

Why divide storage at all?

Keeping the parts separate has real benefits:

  • Update one part safely. A new operating system can be written to its own section without touching your settings.
  • Recover after a bad update. If a device has a recovery copy, it can rebuild the operating system while the bootloader stays untouched.
  • Protect the settings. Because your configuration lives in its own section, replacing the operating system need not erase your passwords and network names.

Erasing the wrong section is dangerous. If the bootloader section is wiped, the device may no longer start at all — there is nothing left to load the operating system. This is why we always make a full backup of the storage before changing anything (covered properly in Part 15). A backup lets you write the working copy back if a step goes wrong.

Examples

  • A phone keeps the system and your photos in separate areas, so a system update does not delete your photos.
  • A home PC can have one drive split so the operating system and your documents sit apart — reinstalling the system need not touch the documents.
  • Our device uses the same idea in miniature: separate sections so the bootloader, Linux, settings and any recovery copy can be handled one at a time.

Try it yourself (safe)

No hardware needed. This is a matching exercise — decide which section holds each thing, then check yourself below.

  1. Your saved Wi-Fi name and password.
  2. The very first program that runs at power-on.
  3. Linux and the programs that run on it.
  4. A spare copy used to repair a broken system.
Self-check: which section holds each item?
  • 1 → the settings section.
  • 2 → the bootloader section.
  • 3 → the operating system section.
  • 4 → the recovery copy (only on devices that have one).

Remember: on your own unit the real names and sizes may differ. You confirm the actual layout by reading it off the device — never by guessing.

Recap

  • Permanent (flash) storage keeps its contents with the power off.
  • It is split into labelled sections called partitions.
  • Typical sections: bootloader, operating system, settings, and sometimes a recovery copy.
  • The split lets parts be updated or repaired separately.
  • Erasing the wrong section can stop the device booting — so back up first.
