Part 9 — Networking Basics (Ethernet, GPON, IP, MAC, DHCP)

How your device finds its address and reaches the Internet over a shared fiber line.

TL;DR

Ethernet is the ordinary wired network — the cable with the wide clip that plugs into computers and routers.

A MAC address is a device's built-in hardware name, fixed at the factory. An IP address is its address on a network, which can change.

DHCP is the helper that hands out IP addresses automatically, so you don't set them by hand.

GPON is the fiber system where many homes share one fiber line back to the Internet company. This device sits at your end of that fiber.

Ethernet: the common wired network

When people say "a network cable", they almost always mean an Ethernet cable. It is the standard way to wire computers, routers and devices together in a home or office. Your fiber device turns the light coming down the fiber into ordinary Ethernet that a normal computer or router can understand.

MAC address: the device's built-in name

Every piece of network hardware is given a MAC address when it is manufactured — a long unique code burned in at the factory, written as six pairs like a1:b2:c3:d4:e5:f6. Think of it as a permanent serial number for the network part. It normally never changes, and it is how a local network tells one physical device from another.

IP address: the device's address on a network

A IP address is where a device can be reached on a particular network — for example 192.168.1.20. Unlike the MAC address, it is not fixed to the hardware: move the device to a different network and it gets a different IP address. If the MAC address is like a person's fingerprint, the IP address is like the street address of wherever they are staying this week.

DHCP: automatic address handout

Setting an IP address by hand for every device would be slow and error-prone. Instead, most networks run a service called DHCP. When a device joins the network it politely asks, "may I have an address?", and the DHCP service hands one out, along with a few other settings it needs. This all happens in a second or two, invisibly. (Services running quietly in the background were Part 8 — DHCP is one of them.)

GPON: how the fiber actually reaches you

GPON is the kind of fiber network many Internet companies use. Its trick is sharing: instead of running a separate fiber all the way from the Internet company to every single home, one fiber leaving their building is split and shared between many homes.

Three pieces make it work:

  • ONT — the box at your end of the fiber. That is what this device is: an "Optical Network Terminal". It turns the light into Ethernet for you.
  • Splitter — a simple part out in the street or a cabinet that splits one fiber into many, so a group of homes can share the line back to the Internet company.
  • OLT — the big unit at the Internet company's end ("Optical Line Terminal") that all those shared fibers connect back to.

So the full path is: your home ↔ ONT ↔ splitter ↔ OLT ↔ Internet. Data flows both ways along that chain — out when you send, back when you receive.

Examples

  • MAC vs IP: your laptop's MAC address is the same in every café; the IP address it is given changes with each café's network.
  • DHCP in action: plug a new laptop into a home router and it "just works" — DHCP quietly handed it an IP address in the background.
  • The shared fiber: a whole street can feed through one splitter back to a single OLT port at the Internet company. Your ONT is your personal end of that shared line.
  • Where this device fits: the word "ONT" simply means the box in your home at the end of the fiber — this Nokia unit.

Try it yourself (safe)

Find your own computer's IP and MAC address. These commands only read and print — they change nothing.

  • Windows: open Command Prompt and type ipconfig /all. Look for "Physical Address" (the MAC) and "IPv4 Address" (the IP).
  • Linux or Mac: open a terminal and type ip addr (on older Macs, ifconfig). The MAC appears after link/ether; the IP after inet.

Run these on your own everyday computer, not on the fiber device. They are read-only and safe. Do not type them into the fiber unit while you are still learning.

Self-check: your laptop shows one MAC address but a different IP address at home and at work. Why?

The MAC address is burned into the laptop's network hardware, so it stays the same everywhere. The IP address is assigned by whatever network you join — usually by that network's DHCP service — so it changes from place to place.

Recap

  • Ethernet is the ordinary wired network your computers and router use.
  • A MAC address is a device's fixed, factory-set hardware name.
  • An IP address is its changeable address on a particular network.
  • DHCP is the service that hands out IP addresses automatically.
  • GPON shares one fiber among many homes: your home ↔ ONT ↔ splitter ↔ OLT ↔ Internet.
  • This device is the ONT — the fiber's end point inside your home.
