Lesson 2 — What's Actually Inside the Nokia G-010G-P?

TL;DR

Your Nokia G-010G-P is a very small Linux computer made for fiber Internet. Instead of a monitor and keyboard, it has an optical fiber connection and Ethernet ports.

PartThink of it as…
CPUThe brain.
RAMThe short-term memory.
Flash storageThe SSD or hard drive.
BootloaderA tiny startup program that loads Linux.
LinuxThe operating system.
ConfigurationSettings like passwords, network names and ISP data.

What happens when you plug it in?

  1. Power reaches the CPU.
  2. The CPU starts running the bootloader.
  3. The bootloader finds Linux in flash storage.
  4. Linux is copied into RAM.
  5. Linux starts running.
  6. The networking programs start.
  7. The fiber connection comes online.

This whole process is called booting.

What is flash storage?

Flash storage permanently holds everything inside the device: Linux, configuration, drivers, programs, logs and recovery software. Even after you unplug the power, this data stays.

What is RAM?

RAM is temporary working memory. When Linux starts, it is copied from flash storage into RAM because RAM is much faster. When power is removed, RAM is completely erased.

What is the bootloader?

The bootloader is the very first software that runs — think of it as the device's starter motor. Many embedded Linux devices use bootloaders such as U-Boot. A bootloader can often start Linux, update it, recover a broken installation, or run diagnostics.

Where is Linux?

Linux lives inside flash storage. The storage is usually divided into several sections — bootloader, Linux operating system, configuration, user data, recovery system. These sections are called partitions.

What does "reverse engineering" mean?

Reverse engineering means figuring out how something works without the original design documents.

For hardware that might mean finding the chips, reading their model numbers, tracing wires on the board, and finding hidden connectors. For software it might mean listing files, reading configuration, studying startup scripts, and looking at the programs. The goal is understanding — not guessing.

What is "root"?

Linux has different permission levels. A normal user can change only some things. The root user can change almost everything: installing software, editing system files, deleting files, changing passwords, configuring hardware.

Always make a complete backup of the device's storage before making changes. If something goes wrong, you have a much better chance of restoring it.

Quick quiz

1. Where does Linux run from once the device is on?

2. What runs first, before Linux?

Self-test: why is a backup so important before touching low-level software?

Because the operating system lives in flash storage. If a change corrupts it, a good backup lets you write the working copy back and recover a device that would otherwise no longer start.
