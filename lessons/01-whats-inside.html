Part 1 — What's Inside the Nokia G-010G-P?

The parts, in plain PC terms, and exactly what happens when you power it on.

TL;DR

Inside the device are the same building blocks as a desktop PC: a brain, some fast memory, permanent storage, and an operating system.

When you plug it in, a tiny starter program wakes up, loads the operating system into fast memory, and the operating system then brings your Internet online.

We stay outside the case this whole lesson — looking and learning only.

The parts, and their PC twins

You already know these parts from an ordinary computer. They just have smaller, quieter versions inside this box.

One quick word to define now, because we'll use it a lot: Linux is the name of the operating system family this device runs. You don't need to know anything about it yet — just that it's the "main software," the same role Windows plays on a laptop.

What happens when you plug it in?

Every start-up follows the same steps, in the same order. This whole sequence is called booting.

  1. Power reaches the CPU and it wakes up.
  2. The CPU runs the bootloader — the tiny starter program.
  3. The bootloader finds the operating system in permanent storage.
  4. It copies the operating system into RAM, because RAM is much faster to work from.
  5. The operating system starts running.
  6. Its networking programs start up.
  7. The fiber connection comes online and your Internet works.

Permanent storage vs RAM

Permanent storage holds everything that must survive being unplugged: the operating system, the settings, the programs, and the recovery software. Pull the plug and it is all still there when you return.

RAM is fast, temporary working memory. The operating system is copied into it at start-up because RAM is quick to work from. The moment power is lost, RAM is completely blank again. This is why nothing you keep long-term lives in RAM.

What is a bootloader?

The bootloader is the very first software to run — the device's starter motor. It is small and single-minded: find the operating system and get it running. On many embedded Linux devices the bootloader is a well-known program called U-Boot, and a bootloader can often also update the system, recover a broken one, or run basic checks. Whether that's exactly the case on your unit is something you'll confirm later by watching the device start.

Where does Linux live?

The operating system lives in permanent storage. That storage is not one big undivided blob — it is split into labelled sections, one for the bootloader, one for the operating system, one for settings, and usually one for a recovery system. Those sections are called partitions: think of a filing cabinet with separate, labelled drawers rather than one big pile.

The exact partition names and sizes vary by model and version. This course teaches you how to read them off your own device later — we won't guess numbers for your specific unit.

What is "root"?

An operating system like Linux has different permission levels. A normal user can change only a few things. The root user is the all-powerful account: it can install software, edit any system file, change passwords, and reconfigure the hardware. Great power, and a great way to break things — which is exactly why we back up first.

What is reverse engineering?

Reverse engineering means working out how something behaves without the original design documents. For hardware, that's finding the chips, reading their part numbers, and locating hidden connectors. For software, it's listing files, reading settings, and studying the start-up scripts. The goal is always understanding — not guessing.

Examples

  • Bootloader as a starter motor: a car engine can't start itself; a small motor gets it turning, then hands off. The bootloader does that for the operating system, then steps aside.
  • RAM as a whiteboard: quick to write on during a meeting, but wiped clean at the end. Permanent storage is the notebook you keep.
  • Partitions as labelled drawers: one drawer for the operating system, one for settings, one for recovery — separated so a mess in one doesn't spill into the others.

Try it yourself (safe)

An observation-only exercise. The case stays shut and nothing gets taken apart.

  1. Look at the outside of your device. Note each port (the sockets you plug things into) — the fiber socket, the Ethernet socket(s), and the power socket — and write down which is which.
  2. Watch the LEDs (the little status lights). Plug the device in and note the order they light up and settle. You are watching the boot steps from the outside: power, then the system starting, then the network coming online.
  3. Read the label again and note anything printed about power (voltage) and the model. Don't act on it — just record it.

Safety: do not open the case. This is a mains-powered device — never open a powered device, and never take this one apart in this lesson. Everything above is done by looking only.

Self-check: the lights change in a set order every time — why?

Because booting always follows the same fixed sequence: power reaches the CPU, the bootloader starts the operating system, and then the networking comes online. The LEDs are simply reporting those same steps from the outside, which is why the pattern repeats on every start-up.

Recap

  • Inside are a CPU, RAM, permanent storage, a bootloader, an operating system, and settings — the same parts as a PC.
  • Booting always follows the same order: power → CPU → bootloader → operating system into RAM → networking → fiber online.
  • Permanent storage survives being unplugged; RAM is fast but wiped every time.
  • Permanent storage is split into labelled partitions, and root is the all-powerful user.
  • You observed your device's ports and LEDs from the outside — no opening the case.
