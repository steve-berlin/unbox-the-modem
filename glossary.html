Glossary

Every technical word this course uses, in plain English. When a part introduces a new word, it's also defined here. Where a term is jargon we normally avoid (like "firmware" or "ROM"), the plain replacement is given.

TermPlain-English meaning
8N1The most common serial setting: 8 data bits, no parity bit, 1 stop bit per byte. You'll usually pick this in your terminal program.
115200A very common serial speed (baud rate). If the text on the console is garbled, the wrong speed is the usual reason.
Anti-staticPrecautions that stop tiny sparks of static electricity from your body damaging chips. See ESD.
Architecture (CPU)The "machine language" family a processor understands, such as ARM, MIPS or x86. Programs must be built for the right one.
ARMA common processor family in small devices and phones. Often different from the x86 in a desktop PC.
BackupA complete copy of the device's storage, saved somewhere safe, so you can restore it if a change goes wrong. The course's golden rule.
Baud rateHow fast bits travel over a serial connection, in bits per second. Both ends must agree (e.g. 115200).
BinaryNumbers written with only 0s and 1s — the form computers work in. Also, a ready-to-run program file.
BitThe smallest piece of data: a single 0 or 1.
Boot / bootingEverything that happens between power-on and the operating system being fully up and running.
Boot logThe stream of text messages a device prints as it starts. You can read it over the console (UART).
Boot loopA fault where the device keeps restarting instead of finishing start-up.
BootloaderThe very first tiny program that runs at power-on. Its job is to find the operating system and start it. Like a car's starter motor.
Brick / brickedA device that no longer starts properly after a bad change. A good backup usually prevents this being permanent.
BuildrootA tool that assembles a complete small Linux system (bootloader + core + programs) for you.
BusyBoxA single small program that provides many common Linux commands at once — popular on tiny devices.
ByteA group of 8 bits. Often one character of text.
CapacitorA small board component that stores and smooths electricity.
ChecksumA short "fingerprint" number calculated from a file. If two copies have the same checksum, they match exactly. Used to verify backups.
ChipA small black component that holds an electronic circuit — the processor, memory, and storage are all chips. See IC.
Command lineA text way of controlling a computer by typing commands, instead of clicking.
CompilingTurning human-written source code into a program the machine can actually run.
ConfigurationThe device's settings: passwords, network names, provider data, and so on.
ConnectorA socket or set of pins on a board where a cable or another part plugs in.
ConsoleA text window into a device that lets you see its messages and type commands. On this hardware you reach it over UART.
ContinuityA multimeter mode that beeps when two points are electrically joined. Used to find ground and trace connections on an unpowered board.
CPUThe processor — the "brain" that runs all the software.
Cross-compilingBuilding a program on your PC that will run on a different kind of processor — the device's.
DaemonA background program that keeps running to provide a service (see Service).
DatasheetThe maker's document describing a chip: what it does and what each pin is for. You find it by searching the chip's printed number.
DHCPThe service that automatically hands out IP addresses to devices on a network.
DirectoryA folder in the file system that holds files and other folders.
DriverSoftware that lets the operating system talk to a specific piece of hardware.
Embedded systemA small computer built into a device to do one job — like this fiber modem.
EmulatorSoftware that pretends to be a different computer, so you can safely run and test a system on your PC. See QEMU.
Environment variableA named setting the bootloader or operating system remembers, such as where to find the operating system.
ESDElectrostatic discharge — a tiny static spark that can silently damage chips. Ground yourself before touching a bare board.
EthernetThe common wired-network connection your computer or router uses.
Fiber / optical fiberA thin glass strand that carries Internet data as pulses of light.
FileA named piece of stored data.
File systemThe way files are organised on storage so they can be found and read.
FirmwareJargon we avoid. Usually means the operating system or the software stored inside the device.
Flash storageThe device's permanent storage (like an SSD). Keeps data even with the power off. We usually just say "permanent storage".
Frame (Ethernet)One packaged chunk of data sent over Ethernet, with the sender's and receiver's hardware names attached.
GND / GroundThe shared "zero" reference point in a circuit. One of the three UART wires. Always connect it.
GPONThe fiber system where many homes share one fiber line back to the provider, using splitters in the street and equipment at both ends.
HardwareThe physical parts you can touch: chips, board, connectors, case.
HexadecimalA number system using 0–9 and A–F, handy for writing bytes compactly.
Host (cross-compiling)The computer you build software on — your PC.
ICIntegrated circuit — a complete circuit packed into one chip.
ImageJargon we avoid. Means a complete copy of the storage or operating system, saved as one file.
initThe very first program the operating system starts; it then starts everything else.
initramfsA small temporary file system loaded into memory early in boot to help get things started.
IP addressA device's address on a network, like 192.168.1.10.
ISPInternet Service Provider — the company that supplies your Internet, and often owns the modem.
JFFS2A file system designed to store writable data safely on flash storage.
Jumper wireA short wire with a plug on each end, used to connect an adapter to pins on a board.
KernelThe core of the operating system that manages the hardware and runs the programs.
LEDA small light on the device that signals status (power, network, activity).
Level shiftingConverting between different voltages (like 5V and 3.3V) so two parts can talk without damage.
LinuxThe operating system running inside this device — the same family that runs many servers and phones.
Logic levelThe voltage a signal uses to mean 0 or 1, commonly 3.3V on these devices. Matching it matters — see voltage.
MAC addressA device's built-in hardware name on a network, unique to that network part.
Mains electricityThe high-voltage power from a wall socket. Dangerous — never open or probe a mains-powered device while it's plugged in.
minicomA terminal program (on Linux) for talking to a device over serial/UART.
MIPSAnother processor family common in networking devices, different from x86.
Mount / mount pointAttaching a storage area into the file-system tree at a folder so its files become reachable.
MTDThe Linux way of presenting the sections (partitions) of flash storage so software can read them.
MultimeterA handheld tool that measures voltage and checks connections (continuity) on a board.
NAND flashA type of permanent storage that packs in lots of data cheaply; common for the main storage.
NOR flashA type of permanent storage that's easy to read directly; often used for the bootloader.
OLTThe provider's equipment at the far end of the fiber that serves many homes at once.
ONTOptical Network Terminal — the device at your end of the fiber. The Nokia G-010G-P is one.
Operating systemThe main software that manages everything and runs the other programs. Here, that's Linux.
Overlay file systemA trick that stacks a small writable layer on top of a read-only base, so the base stays safe.
PartitionOne labelled section of storage. Storage is split into several: bootloader, operating system, settings, and so on.
PCBPrinted circuit board — the flat board the chips and parts are mounted on.
Permission / ownershipThe rule that you only work on a device you own or are clearly allowed to. See Part 19.
PHYThe chip that physically sends and receives network signals (for example over Ethernet).
PIDProcess ID — the number Linux gives each running program.
PinoutA map of which pin on a chip or connector does what.
Power supplyThe part that turns wall power into the low voltages the board needs.
ProcessA program that is currently running. Each has a PID.
PuTTYA terminal program (on Windows) for serial/UART and network connections.
QEMUPopular emulator software that lets you run a built Linux system on your PC instead of risking the real device.
RAMFast temporary memory. The operating system is copied here to run, and it's wiped when power is removed.
Read-onlyStorage or files that can be read but not changed. The operating system area is often kept read-only for safety.
RecoveryA built-in way to restore a device that won't start properly, often through the bootloader.
ResistorA small board component that limits electrical current.
Responsible disclosureReporting a security problem privately to whoever can fix it, before telling the world.
Reverse engineeringWorking out how something works by examining it, without the maker's design documents. Observe, don't guess.
RJ45The rectangular plug and socket used for an Ethernet cable.
ROMJargon we avoid. People often mean the operating system stored in flash — say that instead, unless it's truly read-only memory.
Root (user)The all-powerful user account in Linux that can change almost anything on the system.
Root file systemThe main set of folders and files the operating system runs from. Note: this is different from the root user.
RXThe "receive" wire in a serial link — the side that listens. See TX.
screenA Linux program that, among other things, can act as a terminal for serial/UART.
Serial communicationSending data one bit after another over a wire — how UART works.
ServiceA background program that keeps running to do a job, like handling the network. Also called a daemon.
SilkscreenThe printed white text and outlines on a circuit board that label parts and pins.
SoCSystem on a chip — one chip that combines the processor and many other parts of a computer.
Source codeThe human-readable instructions a program is written in, before it's compiled.
Splitter (GPON)A passive device in the street that shares one fiber among several homes.
SquashFSA read-only, compressed file system often used to hold the operating system on small devices.
SSHA secure way to reach and control a Linux system over a network.
Static electricityStored electrical charge on your body that can spark and damage chips. See ESD / anti-static.
StorageWhere data is kept. Permanent storage keeps it with the power off; RAM does not.
Target (device)In cross-compiling, the machine your program is built to run on — the modem.
TelnetAn older, unencrypted way to reach a Linux system over a network.
Terminal programSoftware on your PC (screen, minicom, PuTTY) that shows the device's console and sends your typing.
Test pointA small exposed pad on a board meant for touching a probe or wire.
tmpfsA file system that lives only in memory and disappears when power is removed.
ToolchainThe set of tools that together turn source code into a program for a particular processor.
TraceA thin copper line on a circuit board that carries a signal between parts.
TXThe "transmit" wire in a serial link — the side that talks. Connect one side's TX to the other side's RX.
UARTA simple wired link over a few wires (TX, RX, GND) that lets you watch a device's messages and type commands as it runs.
UBIFSA modern file system for storing writable data reliably on flash storage.
U-BootA bootloader used by many embedded Linux devices; it often gives a small text prompt on the console.
unameA Linux command that reports system details, including the processor type (`uname -m`).
USB-to-UART adapterA small board that connects your PC's USB port to a device's UART pins so you can use the console.
VoltageThe "pressure" of electricity. Matching voltages (often 3.3V here) matters — the wrong one can damage the device.
x86The processor family in most desktop and laptop PCs — usually different from the device's.
YoctoA larger, flexible tool for building a custom embedded Linux system.
